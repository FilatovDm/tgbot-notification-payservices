# Telegram бот для учёта подписок и уведомлений о списаниях

Бот для Telegram, который помогает вести учёт платных подписок и напоминает о предстоящих списаниях в общем чате. Управлять ботом могут только администраторы, заданные в переменных окружения.

## Технологический стек

- **Язык**: Python 3.10+
- **Telegram**: `aiogram 3.x`
- **База данных**: `aiosqlite` (SQLite, асинхронно)
- **Планировщик задач**: `APScheduler` (`AsyncIOScheduler`)
- **Переменные окружения**: `python-dotenv`
- **Работа с датами**: `python-dateutil`

## Основные возможности

- Хранение списка подписок в SQLite (`subscriptions.db`):
  - название сервиса;
  - сумма списания;
  - дата следующего списания;
  - периодичность (1 / 3 / 6 месяцев, 1 год).
- Управление только для администраторов (по `user_id` из `ADMIN_IDS`):
  - `/start` и главное меню с кнопками:
    - 📋 Показать подписки
    - ➕ Добавить подписку
    - 🔁 Изменить подписку
    - ❌ Удалить подписку
  - Добавление подписки (FSM, пошаговый сценарий).
  - Изменение любой части подписки (название, сумма, дата, периодичность).
  - Удаление подписки с подтверждением.
- Уведомления в общий чат (`GROUP_CHAT_ID`) по расписанию (каждый день в 10:00):
  - за 3 дня до списания;
  - за 1 день до списания;
  - автоматическое перенесение даты следующего списания вперёд согласно периодичности.

## Структура проекта

- `main.py` – инициализация бота, роутеров и планировщика.
- `config.py` – загрузка настроек из `.env`, парсинг `ADMIN_IDS`, `GROUP_CHAT_ID`.
- `database.py` – инициализация БД и CRUD‑функции для таблицы `subscriptions`.
- `handlers.py` – хендлеры команд и кнопок, FSM сценарии добавления/изменения/удаления.
- `scheduler.py` – логика ежедневной проверки подписок и отправки уведомлений.
- `requirements.txt` – список зависимостей.

## Настройка окружения

1. Клонируйте репозиторий:

   ```bash
   git clone git@github.com:FilatovDm/tgbot-notification-payservices.git
   cd tgbot-notification-payservices
   ```

2. Создайте и активируйте виртуальное окружение (пример для Windows PowerShell):

   ```powershell
   python -m venv .venv
   .\.venv\Scripts\activate
   ```

3. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

4. Создайте файл `.env` в корне проекта:

   ```env
   BOT_TOKEN=ваш_токен_бота_из_BotFather
   ADMIN_IDS=123456789,987654321  # ID администраторов через запятую
   GROUP_CHAT_ID=-1001234567890   # ID группового чата, куда слать уведомления
   ```

## Запуск бота

```bash
python main.py
```

Бот начнёт получать обновления (long polling) и запускать планировщик APScheduler.  
В групповом чате с ID `GROUP_CHAT_ID` бот будет отправлять утренние уведомления по подпискам.

## Использование

1. Добавьте бота в нужный групповой чат и убедитесь, что он может писать сообщения.
2. В личном чате админа или в группе (если бот видит `/start` от админа) отправьте команду:

   ```text
   /start
   ```

3. Используйте кнопки:
   - **📋 Показать подписки** – список всех подписок.
   - **➕ Добавить подписку** – пройти сценарий добавления.
   - **🔁 Изменить подписку** – выбрать подписку и поле для изменения.
   - **❌ Удалить подписку** – выбрать и удалить подписку.
