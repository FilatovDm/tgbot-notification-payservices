Ты — Senior Python Developer. Твоя задача — с нуля написать Telegram-бота для учета подписок и уведомления о предстоящих списаниях. Бот будет работать в общем чате, уведомления отправлять туда же, но управлять ботом могут только администраторы.

### Технический стек:
- Язык: Python 3.10+
- Библиотека Telegram: aiogram 3.x
- База данных: aiosqlite (асинхронный SQLite)
- Планировщик задач: APScheduler (AsyncIOScheduler)
- Работа с переменными окружения: python-dotenv
- Работа с датами: python-dateutil (для точного прибавления месяцев к дате)

### Настройки окружения (.env):
Бот должен брать следующие переменные:
- `BOT_TOKEN` (токен бота)
- `ADMIN_IDS` (список ID администраторов через запятую, например: 12345,67890)
- `GROUP_CHAT_ID` (ID общего чата, куда планировщик будет присылать утренние уведомления)

### Архитектура и база данных:
База данных SQLite должна иметь таблицу `subscriptions` со следующими полями:
- `id` (INTEGER PRIMARY KEY AUTOINCREMENT)
- `service_name` (TEXT) - Название сервиса
- `amount` (REAL) - Сумма списания
- `next_payment_date` (DATE) - Дата следующего списания (в формате YYYY-MM-DD)
- `periodicity` (TEXT) - Периодичность (хранить как: '1_month', '3_months', '6_months', '1_year')

### Основной функционал (Требования):
1. Ограничение доступа: Обработчики команд и кнопок должны срабатывать ТОЛЬКО если `message.from_user.id` или `callback_query.from_user.id` находится в списке `ADMIN_IDS`. Остальных пользователей бот должен игнорировать.
2. Команда `/start`: Выводит ReplyKeyboardMarkup с кнопками:
   - 📋 Показать подписки
   - ➕ Добавить подписку
   - 🔁 Изменить подписку
   - ❌ Удалить подписку
3. Флоу добавления подписки (FSM):
   - Запрос названия сервиса.
   - Запрос суммы (число).
   - Запрос даты следующего списания (валидация YYYY-MM-DD).
   - Запрос периодичности. Вывести Inline-кнопки: "1 месяц", "3 месяца", "6 месяцев", "1 год".
   - Сохранение в БД.
4. Флоу изменения подписки (🔁 Изменить подписку) через FSM:
   - Бот выводит список всех подписок в виде Inline-кнопок.
   - Админ выбирает подписку.
   - Бот спрашивает, что именно изменить (Inline-кнопки): Название, Сумма, Дата, Периодичность.
   - Бот запрашивает новое значение, обновляет запись в БД и подтверждает изменение.
5. Флоу удаления (❌ Удалить подписку):
   - Вывод списка подписок Inline-кнопками. 
   - Выбор подписки -> Подтверждение удаления -> Удаление из БД.
6. Кнопка "📋 Показать подписки": 
   - Выводит красивым сообщением список всех сервисов, сумму, дату следующего списания и периодичность. 
   - Сортировка по дате списания (от ближайших к дальним).
7. Система уведомлений (APScheduler):
   - Планировщик запускается каждый день в 10:00.
   - Читает таблицу `subscriptions` и отправляет сообщения в чат с ID `GROUP_CHAT_ID` (брать из конфига).
   - Если до `next_payment_date` ровно 3 дня: "⚠️ Напоминание: Через 3 дня ({next_payment_date}) списание за {service_name} - {amount}".
   - Если до `next_payment_date` ровно 1 день (завтра): "🚨 ВНИМАНИЕ: Завтра ({next_payment_date}) списание за {service_name} - {amount}".
   - Автоматическое обновление даты: Если `next_payment_date` <= сегодня (дата наступила или прошла), бот должен автоматически прибавить к `next_payment_date` нужный интервал (используй dateutil.relativedelta: 1 месяц, 3 месяца, 6 месяцев или 1 год) и обновить запись в БД.

### Структура проекта:
- main.py (инициализация бота, роутеров и планировщика)
- config.py (загрузка переменных из .env, парсинг ADMIN_IDS в list[int])
- database.py (инициализация БД и CRUD функции)
- handlers.py (обработка Reply-кнопок, FSM для добавления, изменения и удаления)
- scheduler.py (логика проверки дат и отправки уведомлений в GROUP_CHAT_ID)
- requirements.txt
